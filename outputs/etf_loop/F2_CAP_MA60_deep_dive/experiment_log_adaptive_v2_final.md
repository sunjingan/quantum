## 零、策略说明：如何选出 Top 5 ETF

### 动量因子计算

核心公式：**Score = 年化收益率 × R²**

分三步：

1. **加权对数线性回归**

取过去 25 个交易日（默认 `lookback_days=25`）的复权价格序列，取对数，做加权线性回归。权重从 1 线性增长到 2——越靠近今天的价格权重越大：

```
y = log(prices)                      # 对数价格
x = [0, 1, 2, ..., 24]               # 时间轴
weights = [1.0, 1.04, 1.08, ..., 2.0]  # 线性递增 (np.linspace(1, 2, 26))
slope = polyfit(x, y, 1, w=weights)   # 加权最小二乘拟合
```

2. **年化收益率**

```
annualized_returns = e^(slope × 250) - 1
```

含义：如果过去 25 天的对数收益率（斜率）持续一整年，对应的年化回报。正值 = 上升趋势，负值 = 下降趋势。

3. **拟合优度 R²**

```
R² = 1 - (加权残差平方和 / 加权总平方和)
```

含义：价格曲线有多「直」。趋势流畅（每天稳步涨/跌）→ R² 接近 1。上蹿下跳 → R² 接近 0。

4. **综合打分**

```
Score = annualized_returns × R²
```

既要涨得多（高年化），也要涨得稳（高 R²）。两个维度都强才是高信号。

**举例**：ETF A 年化 50% 但走势颠簸（R²=0.3）→ Score=15。ETF B 年化 30% 但稳健上涨（R²=0.9）→ Score=27。**选 B**。

### 筛选流程

```
全市场 ETF（44 只 F2_v3 池）
  │
  ├─ Layer 1: 硬过滤（排除）
  │   ├─ 价格数据不足 → 跳过
  │   ├─ 放量滞涨（成交量放大 + 高收益 → 潜在顶部） → 跳过
  │   ├─ RSI 超买 + 价格低于 MA5 → 跳过
  │   ├─ 短动量（10 日年化 < 0%） → 跳过
  │   ├─ [可选] 动态回溯窗口、Wyckoff 派发区、波动率过滤、趋势过滤等
  │
  ├─ Layer 2: 动量评分
  │   └─ Score = 加权对数回归年化 × R²
  │
  ├─ Layer 3: 软惩罚（降分不排除）
  │   ├─ 3 日内有单日跌超 3% → Score 归零
  │   ├─ MA60 过热（价格/MA60 ≥ 1.14）→ Score × 0.5
  │   ├─ [可选] 溢价惩罚、反转过滤等
  │
  ├─ Layer 4: 排名 + 筛选
  │   ├─ Score 降序排列
  │   ├─ Switch Score Margin（持仓 ETF 对挑战者有额外分数容忍）
  │   └─ 取 Top N（默认 5）
  │
  └─ 输出：Top 5 ETF → 等权（或 ScoreW 加权）
```

## 十一、实验补充：分数阈值在 Baseline 上的独立测试

### 背景

之前所有分数阈值实验都叠加在 Adapt15 上。这里剥离自适应持仓，单独测试三种阈值方法在固定 5 只 Baseline 上的效果。

### 分数分布（2013-2026，3943 笔买入信号）

| 分位数 | 值 |
|---:|---:|
| P10 | 0.03 |
| P25 | 0.17 |
| P50 | 0.63 |
| P75 | 1.77 |
| P90 | 4.54 |
| P95 | 5.26 |
| P99 | 24.00 |

**Top5 / Top1 比值中位数 = 0.15**。88% 的交易日 Top5 不到 Top1 的 50%。75% 不到 30%。

### 三种阈值方法

| 方法 | 逻辑 |
|---|---|
| Dynamic | `min_score = Top1_score × ratio`（ratio = 0.1~0.7） |
| Fixed | `min_score = X`（X = 0.1~1.0，仅当 N<5 时触发） |
| Rolling 252d P60 | 过去 252 个交易日所有 Top ETF 分数的 P60 分位数 |

### 结果

| 方法 | 参数 | 年化 | DD | ΔAnn |
|---|---:|---:|---:|---:|
| Baseline | — | 29.01% | -18.98% | — |
| Dynamic | ratio=0.1 | 29.01% | -18.98% | 0% |
| Dynamic | ratio=0.2~0.7 | 同上 | 同上 | 0% |
| Fixed | th=0.1~1.0 | 29.01% | -18.98% | 0% |
| **Rolling 252d P60** | — | **32.47%** | -25.31% | **+3.5%** |

### 分析

**Dynamic Threshold 为何无效**：Top5/Top1 比值中位数 0.15。设 ratio=0.5 会砍掉第 2-5 名。设 ratio=0.1 等于不过滤。没有有效中间档位。

**Fixed Threshold 为何无效**：引擎代码逻辑是 `if effective_holdings < params.holdings_num → 启用阈值`。Baseline 固定 5 只 → 条件永不成立 → 阈值永远不触发。

**Rolling P60 勉强有效**：+3.5% 年化但 DD 恶化 6.3pp。代价超过收益。

### 结论

分数阈值本质上不是独立过滤，而是降仓时的辅助工具。仅当仓位减少时（如自适应持仓降到 N<5），阈值才有意义——帮你在有限的仓位里挑最强信号。

有意义的阈值范围（基于日 Top-10 分数分布）：

| 阈值 | 保留率 |
|---:|---:|
| 0.3 | 67% |
| 0.5 | 56% |
| 1.0 | 41% |
| 1.5 | 31% |

建议选择 0.5 作为降仓时的默认阈值（保留过半候选，同时砍掉最弱的 44%）。

### 代码

- `use_dynamic_score_threshold` + `dynamic_score_threshold_ratio`：Top1 × ratio 法（不推荐）
- `adaptive_score_threshold`：Fixed 阈值（仅降仓时有效）
- `use_rolling_score_threshold` + `rolling_score_window`：滚动 P60 法（推荐，但单独效果有限）
- `score_history`：在主循环前初始化为 `[]`，每轮追加 Top 20 分数，维持 252 日窗口

## 十二、Tier 截断 Bug 修正 + 修正后完整结果

### Bug 描述

`bench_20d_ret` 模式中，配置化分级阈值存在截断 bug：

```python
tiers_n = [5, 4, 3, 2, 1, 0]       # 6 元素（含现金兜底）
tiers_ret = [0.05, 0.02, 0.00, -0.03, -0.06]  # 5 阈值

if len(tiers_n) > len(tiers_ret):    # 6 > 5 → True
    tiers_n = tiers_n[:5]             # [5, 4, 3, 2, 1] ← 现金 0 被截断！
```

**影响**：策略永远无法空仓（N≥1），导致 DD 从 -25.44% 飙到 -38.8%。所有实验 11-11（TIER 系列）均受此 bug 影响。

**修正**：
```python
while len(tiers_n) < len(tiers_ret) + 1:
    tiers_n.append(0)    # 补足现金兜底
```

## 十四、最终结论

### 推荐默认配置

```python
# 自适应持仓（15d 窗口 + 沪深300 基准 + 原分级）
use_market_adaptive_holdings = True
adaptive_mode = "bench_20d_ret"
adaptive_window = 15
benchmark = "sh000300"
# 成本：3.5bp/边，7bp 双边
```

### 不推荐

- **ScoreW**：叠加后 DD 恶化（-27.81% vs -25.44%）。自适应持仓已降仓，ScoreW 再集中权重 → 双重集中 → DD 放大
- **Wide A/B**：降收益换 DD 改善幅度太小（-21.8% vs -25.4%，仅改善 3.6pp，年化却掉了 3.9pp）
- **分数阈值**：独立无效，仅在降仓时有辅助价值

### 待改进

- **DD 仍偏高**（-25.4% vs Baseline -19.0%）。需要非仓位维度的 DD 控制：止损线、板块上限、波动率目标仓位
- **N<5 占比过高**（79%）。原分级对 15d 来说偏紧。但放宽分级会降收益，tradeoff 无法避免
- **2023-2024 是弱年**（-3.1% vs Baseline）。动量策略的自然弱周期，不建议针对性调整

### 修正项汇总

| 项目 | 修正前 | 修正后 |
|---|---:|---:|
| 成本 | 2.5bp/边（误报） | **3.5bp/边** |
| DD（Adapt15 Cur） | -38.8%（tier bug） | **-25.44%** |
| qlib 基准 | 仅 HS300 | HS300 + CSV 回退（ZZ500/CYB 等） |
| 引擎 | 分级阈值硬编码 | 配置化 `adaptive_tiers_ret/n` |

## 十九、V3 实验：Ns Exposure + Holdings 分离

### 背景

三张诊断表揭示了 Current 的结构性缺陷：**N 同时控制「风险暴露」和「集中度」**。市场弱 → N 降 → 持仓集中 + 暴露减少。暴露减少是对的，但集中度增加是危险的。

Current 在 2024 年不是满仓扛跌，而是已经降仓到 N=1/2、在弱势中反复换仓（90 日 8 次），形成集中踩踏。Wide A 缓解了集中问题但牺牲了 2018 防守。

### V3 核心思想

把 `N` 拆成两个独立变量：

| 变量 | 含义 | 原有方式 |
|---|---|---|
| `target_exposure` | 总仓位比例 | 原来通过降低 N 隐式控制 |
| `target_holdings` | 持仓 ETF 数量 | 原来就是 N |

新规则：市场弱 → 总仓位降低（留现金），但持仓数不随 N 下降过度减少。

### 引擎改动

**1. EngineParams 新增参数**
```python
adaptive_tiers_exposure: str = "1.00,1.00,1.00,1.00,1.00,0.00"
```
第 146 行，与 `adaptive_tiers_ret`/`adaptive_tiers_n` 并列。

**2. bench_20d_ret 模式新增曝光计算**
```python
tiers_exp = [float(x) for x in params.adaptive_tiers_exposure.split(',')]
while len(tiers_exp) < len(tiers_ret) + 1:
    tiers_exp.append(0.0)
target_exposure = tiers_exp[-1]  # 兜底
for i, t in enumerate(tiers_ret):
    if ret_20d >= t:
        target_exposure = tiers_exp[i]
        break
```

**3. 买入段改为曝光控制**
```python
# 原：per_slot = total_value / n_active
# 新：
investable = total_value * target_exposure
per_slot = investable / n_active
```

`target_exposure = 1.0` 时行为与原版完全一致。默认值确保不改动任何旧实验。
