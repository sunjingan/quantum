## Code Integrity Verification
### 1. 复权因子处理
- `signal_close` 由 `pct_chg` 累计乘积构建 -> 天然连续复权，不受除权除息跳变影响
- 原始 `close` 仅在初始化时作为基准锚点使用（`_build_adjusted_close` 中 `first_close`）
- `signal_open/high/low/vwap` 均通过 `price = signal_close x (raw_field / raw_close)` 比值对齐
### 2. 停牌处理
- 数据读取阶段: `pct_chg.fillna(0.0)` — 停牌日涨幅填0，保证连续曲线无断点
- 估值阶段: `signal_close.ffill()` — forward fill 填补缺失日，维持持仓估值连续
- 执行阶段: `execution_price()` 在当日无价格时返回 NaN -> `execute_sell/buy` 检查并跳过
- 实际效果: 停牌期间持仓不动（估值 forward fill），复牌后按开盘价执行调仓
### 3. 未来函数检查
- 信号计算: `store.price_series(code, date, lookback)` -> 使用 `loc[:date]` 严格截断
- 动量打分: `get_ranked_etfs(store, signal_date, params)` -> 仅使用 `signal_date` 之前数据
- 执行延迟: 信号日收盘(signal_date)打分 -> 次日(exec_date)开盘价成交 -> 无 look-ahead
- 风控止损: 止损价基于 `entry_prices`（买入时记录），当天信号日收盘触发 -> 次日开盘执行
### 4. 实际交易审计
- 信号日 >= 执行日的违规交易: 0 笔 (应为0)
- 信号->执行延迟: 平均 1.0 天, 最小 1 天, 最大 6 天

## 9. 国内券商量化通道手续费参考
- **佣金**: 量化机构客户可谈至 **万0.5-万1** (0.005%-0.01%)。散户通常在万1.5-万2.5
- **滑点**: ETF 流动性好的 (日成交 > 1亿) 滑点约万0.5-万1。流动性差的 (> 5000万) 可能万2-万5
- **印花税**: ETF **免印花税** (这是 ETF 相比股票的巨大优势，股票印花税万5)
- **单边总成本**: 佣金+滑点 ≈ **万1-万2** (0.01%-0.02%)。保守估计万3
- **量化通道**: QMT、Ptrade、恒生PT等主流量化通道。需要关注通道容量和报单速度
